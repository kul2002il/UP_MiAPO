Содержание

Введение

Цель

Получение и закрепление навыков по изучению и анализу предметной области, разработке прецедентов, диаграмм вариантов использования, деятельности, классов, состояний, последовательности системы и структурных схем базы данных.

Задание

Вариант 11 — Реализуйте информационную систему «Журнал класса» Обязательные компоненты: ФИО ученика, ФИО учителя, название предмета, оценка, название темы по предмету, итоговая оценка.

Описние предметной области

Журнал класса — это виртуальная площадка, которая служит для обеспечения возможности учителям выставлять оценки и выдавать задания ученикам, автоматический подсчёт среднего балла.

Виды деятельности

  1. Управление оброзовательными дисциплинами
  2. Выдача задания
  3. Выставление оценок

Для данной предметной области можно выделить следующих актеров:

Актер Общее описание
Пользователь Неавторизированный ученик, учитель, администратор. Или просто гость.
Ученик Пользователь, который хочет узнать предметы, задания, оценки.
Учитель Пользователь, который обращается к системе для выдачи заданий, оценок.
Администратор Сотрудник, проверяющий корректное занесение данных в БД. Также отвечает за регистрацию пользователей, предоставление им прав доступа, производит управление учебными дисциплинами в журналах.
БД Хранилище данных, в которое заносятся необходимая информация.

Система предоставляет следующие возможности:

  1. Пользователь может посмотреть список преметов и задания предмета.
  2. Ученик может посмотреть, помимо предметов и заданий, свои оценки.
  3. Учитель может добаавлять задания и ставить оценки.
  4. Администратор созданиёт, редактирует, удаляет предметы, пользователей (редактирование пользователя — изменение прав пользователя). В особых случаях может редактировать задания и оценки.
  5. БД является внешним хранилищем. Ничего не отправляет, только предоставляет запрашиваемую информацию.
